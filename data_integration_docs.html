Ombaa Data Integration Documentation

Guide for integrating scraped data into the Ombaa directory site

Overview

This document explains how the scraped solar park data has been processed and integrated into the Ombaa directory website. The integration follows a modular approach that allows for easy updates and expansion as more data becomes available.

48 solar parks from the Netherlands have been successfully processed and integrated into the directory.

Data Processing

The raw data from Apify Google Maps Scraper has been processed using a Python script that:

  • Filters entries to identify genuine solar parks
  • Extracts and normalizes location information
  • Estimates park size in hectares when not explicitly provided
  • Determines vegetation types based on available descriptions
  • Maps entries to regions/provinces within the Netherlands
  • Structures the data according to the Ombaa directory schema

Data Schema

{
  "name": "Solar Park Name",
  "location": "Full Address",
  "country": "Netherlands",
  "region": "Province/Region Name",
  "total_hectares": 25.5,
  "vegetation_type": "Mixed grass",
  "contact_email": "contact@example.com",
  "contact_phone": "+31123456789",
  "website": "https://example.com",
  "coordinates": {
    "latitude": 52.1234,
    "longitude": 5.6789
  }
}

Integration Method

The processed data has been integrated into the directory using a JavaScript module that:

  • Loads the JSON data file asynchronously
  • Dynamically creates HTML elements for each solar park
  • Implements pagination with "Load More" functionality
  • Supports filtering by country, region, and listing type
  • Maintains the consistent design of the directory

File Structure

  • /data/processed_solar_parks.json - The processed data file
  • /js/directory-data.js - JavaScript for loading and displaying the data
  • /directory.html - The main directory page that loads the data

Deployment Instructions

To deploy the directory with the integrated data:

  1. Create a /data directory in your web root
  2. Copy processed_solar_parks.json to the /data directory
  3. Include the directory-data.js script in your directory.html page
  4. Ensure the HTML contains elements with the correct IDs:
    • solar-farms-list - Container for solar farm listings
    • load-more-farms - Button for loading more listings
    • apply-filters - Button for applying filters

Note: The contact functionality requires the contact modal implementation from the original directory.html file.

Extending the Integration

To add more data sources or regions:

  1. Process new data using the same Python script or a modified version
  2. Combine the JSON outputs or keep them as separate files by region
  3. Update the JavaScript to load multiple data files if needed
  4. Enhance the filtering functionality to support the new data sources

For shepherd data integration, follow the same pattern but create a separate JSON file and update the JavaScript to load and display shepherd listings in the appropriate container.
